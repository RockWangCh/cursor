# CCH (Customizable Contraction Hierarchies) 寻路算法详解

## 1. 算法概述

CCH (Customizable Contraction Hierarchies) 是一种高效的最短路径查询算法，特别适用于大规模路网的路径规划。它是 Contraction Hierarchies (CH) 算法的扩展版本，具有以下特点：

- **预处理阶段**：构建层次化的图结构
- **查询阶段**：利用层次结构快速找到最短路径
- **可定制性**：支持动态权重更新，无需完全重新预处理

## 2. 核心思想

### 2.1 节点重要性排序
CCH算法首先对图中的所有节点按重要性进行排序。重要性通常基于：
- 节点度数（连接的边数）
- 地理位置的中心性
- 交通流量等因素

### 2.2 节点收缩 (Contraction)
按照重要性从低到高的顺序，逐个"收缩"节点：
1. 移除当前节点
2. 为了保持最短路径的正确性，添加"快捷边"(shortcuts)
3. 快捷边连接原本通过被移除节点相连的节点对

### 2.3 层次结构构建
收缩过程创建了一个层次化的图结构：
- 底层：原始图的所有节点和边
- 上层：重要节点和快捷边
- 查询时从底层向上层搜索，然后从上层向下层搜索

## 3. 算法优势

### 3.1 相比传统Dijkstra算法
- **时间复杂度**：从O(V²)降低到O(log V)
- **空间效率**：预处理后查询速度极快
- **适用场景**：大规模路网的重复查询

### 3.2 相比标准CH算法
- **可定制性**：支持边权重的动态更新
- **灵活性**：可以处理时间相关的权重
- **实用性**：更适合实际应用场景

## 4. 算法流程

### 4.1 预处理阶段
```
1. 计算所有节点的重要性得分
2. 按重要性对节点排序
3. 从最不重要的节点开始收缩：
   a. 计算通过该节点的所有最短路径
   b. 添加必要的快捷边
   c. 从图中移除该节点
4. 构建最终的层次图
```

### 4.2 查询阶段
```
1. 双向搜索初始化：
   - 前向搜索：从起点开始，只使用"向上"的边
   - 后向搜索：从终点开始，只使用"向下"的边
2. 交替执行前向和后向搜索
3. 当两个搜索相遇时，找到最短路径
4. 路径重构：展开快捷边，得到实际路径
```

## 5. 复杂度分析

- **预处理时间**：O(V log V + E)
- **预处理空间**：O(V + E)
- **查询时间**：O(log V)
- **查询空间**：O(log V)

其中 V 是节点数，E 是边数。

## 6. 实际应用

CCH算法广泛应用于：
- GPS导航系统
- 交通路径规划
- 物流配送优化
- 游戏中的NPC寻路
- 网络路由优化

## 7. 算法变种

- **Time-dependent CCH**：处理时间相关的权重
- **Multi-level CCH**：多层次的层次结构
- **Parallel CCH**：并行化实现
